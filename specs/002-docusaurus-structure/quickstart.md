# Quickstart: Docusaurus Structure for Physical AI Book

**Feature**: 002-docusaurus-structure
**Date**: 2025-01-07
**Status**: Draft

## Overview

This quickstart guide provides the essential steps to set up and configure a Docusaurus-based documentation site for the Physical AI & Humanoid Robotics book. It covers installation, configuration, and validation steps needed to create a proper academic book structure with GitHub Pages deployment.

## Prerequisites

Before setting up the Docusaurus structure, ensure you have:

- Node.js v18.0 or higher installed
- npm (comes with Node.js) or yarn package manager
- Git for version control
- PowerShell or command prompt access
- A GitHub account (for deployment)

## Installation Steps

### 1. Navigate to Project Directory
```powershell
cd D:\Quarter_04\hackathon_phase_01\physical-ai-and-humanoid-robotics-book
```

### 2. Create Book Directory
```powershell
mkdir book
cd book
```

### 3. Initialize Docusaurus Project
```powershell
npx create-docusaurus@latest . classic --typescript=ts
```

### 4. Install Additional Dependencies (if needed)
```powershell
npm install
```

## Front Matter Standards

For each document file, include the following front matter:

```markdown
---
title: Your Chapter Title
sidebar_label: Short Label
sidebar_position: 1
description: Brief description of the chapter content
keywords: [keyword1, keyword2, keyword3]
---
```

For chapter files, ensure to include:
- Learning objectives at the beginning
- References section at the end
- Proper heading hierarchy (H1, H2, H3, H4)

## GitHub Pages Deployment

### 1. Deployment Configuration

In your GitHub repository settings:
1. Go to Settings > Pages
2. Set Source to "Deploy from a branch"
3. Select `gh-pages` branch and `/` folder

### 2. Deployment Command

To manually deploy:
```powershell
npm run deploy
```

This command will:
1. Build the site
2. Push the build output to the `gh-pages` branch
3. GitHub will automatically serve the site

## Troubleshooting

### Common Issues

1. **Build fails with "Cannot find module" errors:**
   - Run `npm install` to ensure all dependencies are installed
   - Verify Node.js version is 18+ using `node -v`

2. **Local development server doesn't start:**
   - Check that you're in the `book` directory
   - Ensure no other processes are using port 3000

3. **GitHub Pages deployment fails:**
   - Verify `baseUrl`, `organizationName`, and `projectName` in `docusaurus.config.js`
   - Ensure repository is public if using GitHub Pages

### Windows-Specific Issues

1. **PowerShell execution policy prevents script execution:**
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```

2. **Long file paths in npm:**
   - Enable long path support in Windows
   - Or clone the repository in a shorter path
