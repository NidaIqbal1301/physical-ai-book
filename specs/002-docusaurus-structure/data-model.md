# Data Model: Docusaurus Structure for Physical AI Book

**Feature**: 002-docusaurus-structure
**Date**: 2025-01-07
**Status**: Draft

## Overview

This document defines the conceptual data model for the Docusaurus-based documentation structure of the Physical AI & Humanoid Robotics book. The model outlines the entities, attributes, and relationships that will be implemented in the Docusaurus site structure.

## Core Entities

### Docusaurus Site
- **Description**: The complete website structure built using Docusaurus framework
- **Attributes**:
  - Configuration: docusaurus.config.js settings
  - Base URL: Deployment path (e.g., /physical-ai-book/)
  - Title: Site title and tagline
  - Favicon: Site icon
  - Theme: Visual styling and customization
- **Relationships**: Contains multiple Book Modules; has Navigation Structure

### Book Module
- **Description**: Major sections of the book (ROS 2, Digital Twin, AI-Robot Brain, VLA)
- **Attributes**:
  - Name: Module name (e.g., "The Robotic Nervous System")
  - ID: Unique identifier for navigation (e.g., "module-1-ros2")
  - Position: Order in site navigation
  - Description: Brief overview of module content
  - Learning Outcomes: List of objectives for this module
- **Relationships**: Contains multiple Book Chapters; belongs to Docusaurus Site

### Book Chapter
- **Description**: Individual chapters within each module
- **Attributes**:
  - Title: Chapter title
  - ID: Unique identifier within module
  - Position: Order within parent module
  - Front Matter: title, sidebar_label, sidebar_position, description
  - Content Type: Markdown or MDX
  - Learning Objectives: Specific learning goals for this chapter
  - References: List of citations used in the chapter
- **Relationships**: Belongs to a Book Module; may link to other Book Chapters

### Navigation Structure
- **Description**: The hierarchical organization of modules and chapters in sidebar
- **Attributes**:
  - Type: Sidebar organization type (autogenerated, manual)
  - Collapsible: Whether sections can be collapsed
  - Link Type: How links are structured (pathname, doc)
  - Order: The sequence of items in navigation
- **Relationships**: Organizes multiple Book Modules and Book Chapters

### Document Front Matter
- **Description**: Metadata for each document including title, sidebar_label, and sidebar_position
- **Attributes**:
  - Title: The page title
  - Sidebar Label: Text displayed in sidebar navigation
  - Sidebar Position: Order position in sidebar
  - Description: SEO description for the page
  - Keywords: SEO keywords for the page
- **Relationships**: Applied to Book Chapters

### Content Flow Relationships
- Book Module → Index Page (provides overview of module)
- Book Module → Multiple Book Chapters (contains detailed content)
- Book Chapter → Learning Objectives (defines learning goals)
- Book Chapter → References (citations for academic rigor)
- Book Chapter → Internal Links (cross-links to related content)

## Configuration Entities

### Docusaurus Configuration
- **Description**: Central configuration file for Docusaurus site
- **Attributes**:
  - Title: Site name
  - Tagline: Site description
  - URL: Website URL
  - Base URL: Base path for deployment
  - Organization Name: GitHub organization
  - Project Name: GitHub repository name
  - Themes: Enabled themes and plugins
  - Presets: Selected Docusaurus presets
- **Relationships**: Configures Docusaurus Site behavior

### Sidebar Configuration
- **Description**: Navigation structure for the site
- **Attributes**:
  - Module Grouping: How modules are organized
  - Chapter Ordering: Sequence of chapters within modules
  - Collapsible Sections: Which sections can be collapsed
  - Link Labels: Text displayed for navigation items
- **Relationships**: Defines Navigation Structure

## Content Standards

### Markdown Structure
- Heading Hierarchy (H1 to H4): Follows proper semantic structure
- Learning Objectives Placement: At the beginning of each chapter
- References Section: At the end of each chapter
- Internal Linking: Using Docusaurus linking conventions

### Academic Requirements
- Citation Format: APA style
- Fact Traceability: All claims must have verifiable sources
- Plagiarism Policy: 0% tolerance
- Readability: Flesch-Kincaid Grade 10-12

## Deployment Configuration

### GitHub Pages Settings
- **Attributes**:
  - Build Command: `npm run build`
  - Output Directory: `build/`
  - Base URL Configuration: Path adjustment for GitHub Pages
  - Branch: Deployment branch (typically gh-pages)

## Validation Criteria

### Structural Requirements
- Each Book Module has clear navigation path
- Sidebar properly reflects module hierarchy
- All internal links are valid
- Content follows proper heading structure

### Academic Compliance
- All chapters include learning objectives
- All chapters have references section
- All claims can be traced to authoritative sources
- Content follows academic clarity standards

### Technical Requirements
- Site builds without errors
- All pages load correctly
- Navigation works properly
- Mobile responsiveness implemented
- SEO best practices followed

## Dependencies & Constraints

### Technical Dependencies
- Node.js v18+ for Docusaurus compatibility
- npm or yarn for dependency management
- Git for version control
- GitHub for hosting and deployment

### Content Constraints
- Total word count: 5,000-7,000 words across all modules
- All claims must be traceable to authoritative sources
- Zero plagiarism tolerance
- Docusaurus Markdown format for GitHub Pages deployment
